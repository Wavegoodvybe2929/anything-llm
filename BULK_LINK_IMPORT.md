# Bulk Link Import with DocLinks CLI

This guide explains how to import multiple documentation links into AnythingLLM Docker using the [DocLinks CLI](https://github.com/yourusername/doclinks) tool. This allows you to vectorize entire documentation sites efficiently.

## Overview

The DocLinks CLI tool crawls documentation websites and outputs a `.txt` file containing discovered URLs (one per line). You can then import this file directly into AnythingLLM to bulk-process all the links.

## Prerequisites

1. **AnythingLLM Docker** running with the document processor online
2. **DocLinks CLI** installed on your local machine

### Installing DocLinks

```bash
# From source (requires Rust 1.70+)
git clone https://github.com/yourusername/doclinks.git
cd doclinks
cargo install --path .

# Or with Cargo
cargo install doclinks
```

## Step 1: Generate Links with DocLinks

Use DocLinks to crawl your target documentation site and export the URLs to a `.txt` file:

```bash
# Basic crawl - outputs to stdout
doclinks crawl https://docs.example.com/

# Save to a .txt file
doclinks crawl https://docs.example.com/ --output docs-links.txt

# Limit depth and pages for faster crawling
doclinks crawl https://docs.example.com/ --max-depth 3 --max-pages 100 --output docs-links.txt

# For better coverage, use hybrid mode with SearXNG
doclinks hybrid https://docs.example.com/ --instance http://localhost:8080 --output docs-links.txt
```

### Output Format

The `.txt` output contains one URL per line:

```
https://docs.example.com/
https://docs.example.com/getting-started
https://docs.example.com/installation
https://docs.example.com/configuration
https://docs.example.com/api/overview
...
```

## Step 2: Import Links into AnythingLLM

1. **Open AnythingLLM** in your browser (typically `http://localhost:3001` for Docker)

2. **Navigate to your workspace** and click the settings icon

3. **Open the Documents section** in the workspace settings modal

4. **Click "Import links from file"** button below the link input field

5. **Select your `.txt` file** generated by DocLinks

6. **Wait for processing** - AnythingLLM will scrape each URL and add it to your documents

After import, you'll see a success message indicating how many links were successfully processed and how many failed (if any).

## Tips

### Filtering Links Before Import

Edit the `.txt` file to remove any unwanted URLs before importing:

```bash
# Remove lines containing certain patterns
grep -v "changelog\|release-notes" docs-links.txt > filtered-links.txt

# Keep only documentation pages
grep "/docs/\|/guide/\|/api/" docs-links.txt > filtered-links.txt
```

### Handling Large Link Lists

For documentation sites with thousands of pages, consider:

1. **Splitting into smaller batches**:
   ```bash
   split -l 100 docs-links.txt batch-
   ```

2. **Limiting crawl scope**:
   ```bash
   doclinks crawl https://docs.example.com/api/ --max-pages 500 --output api-docs.txt
   ```

### Troubleshooting

| Issue | Solution |
|-------|----------|
| "Document Processor Unavailable" | Ensure the collector service is running in Docker |
| Links failing to process | Check if the URLs are accessible and not behind authentication |
| Slow processing | Process in smaller batches or increase system resources |

## Example Workflow

```bash
# 1. Crawl Python documentation
doclinks crawl https://docs.python.org/3/ \
  --max-depth 4 \
  --max-pages 500 \
  --output python-docs.txt

# 2. Review and filter (optional)
wc -l python-docs.txt  # Check number of links
head -20 python-docs.txt  # Preview first 20 links

# 3. Import into AnythingLLM via the UI
# - Open workspace settings
# - Go to Documents tab
# - Click "Import links from file"
# - Select python-docs.txt
```
